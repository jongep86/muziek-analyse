# Plan: YouTube-import voor Muziek-Analyse

## Context

Gebruikers moeten nu handmatig audio downloaden van YouTube en converteren voor ze een nummer kunnen analyseren. Issue #1 vraagt om een YouTube URL-invoer in de webapp zodat yt-dlp de download/conversie automatisch afhandelt en de analyse-pipeline direct start.

## Aanpak

### 1. Dependency toevoegen
**`requirements.txt`** — voeg `yt-dlp` toe.

### 2. Config uitbreiden
**`config.py`** — voeg `YOUTUBE_DOWNLOAD_DIR = os.path.join(BASE_DIR, 'tracks')` toe.

### 3. Nieuwe utility module `youtube.py`
Losse module met:
- `validate_youtube_url(url)` — regex check voor youtube.com/youtu.be patterns
- `extract_info(url)` — haal metadata op (titel, duur) via `yt_dlp.YoutubeDL.extract_info(download=False)` voor snelle validatie
- `download_audio(url, output_dir, progress_callback)` — download + converteer naar mp3 via yt-dlp als Python library. Geeft dict terug met `file_path`, `title`, `duration`. Gebruikt `restrictfilenames=True` voor veilige bestandsnamen, `noplaylist=True`, postprocessor naar mp3 192kbps. Progress hook vertaalt download % naar callback-berichten.

### 4. Database-migratie
**`models.py`**:
- Voeg `source TEXT DEFAULT 'local'` en `youtube_url TEXT` kolommen toe via `ALTER TABLE` migratie in `init_db()`
- Update `create_track()` met optionele `source` en `youtube_url` parameters
- Voeg `update_file_path(track_id, file_path, file_size)` toe om pad bij te werken na download

### 5. Taak-queue uitbreiden
**`tasks.py`**:
- Refactor queue-items van tuples naar dicts: `{"type": "analyse"|"youtube", ...}`
- Bestaande `enqueue()` blijft backward compatible (maakt intern dict aan)
- Nieuwe `enqueue_youtube(track_id, youtube_url, output_dir, db_path)`
- Nieuwe `_run_youtube_import()`:
  1. Status → `'downloading'`, roept `download_audio()` aan met progress callback
  2. Update `file_path` in DB na download
  3. Status → `'analysing'`, roept direct `_run_analysis()` aan
  4. Bij fouten → status `'error'` met foutmelding

### 6. Nieuwe route
**`app.py`**:
- `POST /youtube` route: valideer URL, haal metadata op (synchroon, ~1-2s), maak track aan, enqueue youtube-taak
- Voeg `'downloading'` toe aan `active_tracks` filter (regel 46)
- `os.makedirs(YOUTUBE_DOWNLOAD_DIR)` bij startup
- Bij delete van youtube-track: verwijder ook het gedownloade bestand van disk

### 7. UI-aanpassingen
**`templates/index.html`** — YouTube URL-formulier boven de zoekbalk:
```html
<div class="youtube-import">
  <form method="POST" action="/youtube" class="youtube-form">
    <input type="url" name="youtube_url" placeholder="Plak een YouTube URL..." required>
    <button type="submit" class="btn btn-primary">Importeer van YouTube</button>
  </form>
</div>
```
Plus badge-stijl voor `downloading` status.

**`static/style.css`** — minimale styling voor `.youtube-import` / `.youtube-form` (flexbox row).

**`static/app.js`** — geen wijzigingen nodig; bestaande polling toont al arbitraire step-berichten.

## Verificatie
1. `pip install -r requirements.txt` (yt-dlp geinstalleerd, ffmpeg moet beschikbaar zijn)
2. Start app met `python app.py`
3. Plak een YouTube URL in het formulier op de hoofdpagina
4. Controleer dat de voortgang zichtbaar is (downloading → analysing → done)
5. Bekijk het resultaat op de track detail pagina
6. Test foutafhandeling: ongeldige URL, onbeschikbare video
7. Test verwijderen van YouTube-track (bestand moet ook van disk verdwijnen)
8. Test dat bestaande lokale analyse-flow ongewijzigd werkt
